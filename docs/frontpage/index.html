MyVariant.info Data Restore Instructions

Once the Docker container is running, you can initiate the data restore script as follows:

  1. docker exec -d [container_name] /home/biothings/es_restore

    This step will initiate a restore of ~ 1TB of data from our AWS S3 bucket. Depending on your system configuration, it can take many hours to complete.

  2. To monitor the data restore progress, you can open your web browser and point it to: http://localhost:9200/_plugin/head. This page is an Elasticsearch management tool. Each index (variant_hg19_20170517 and variant_hg38_20170517) will eventually be shown here, along with 20 boxes numbered 0-19 underneath each. Each of these boxes represents a shard of the data. When a shard has completed recovery and is searchable, its box will turn from yellow to green. Once all 40 shards (in both indices) turn green, the entire database is searchable.

What to do next

Restore Variant data

Once the Docker container is running, you can initiate the data restore script as follows: 

docker exec -ti CONTAINER /home/biothings/es_restore.sh

Monitor data restoration progress

The last step will initiate a restore of ~ 1TB of data from our AWS S3 bucket. Depending on your system configuration, it can take quite a whole, varying from over-night to two-days. You can monitor the progress at "http://docker_host:9201/_plugin/head". Once all blocks are turning green and the "cluster health" on the top displays "green (40 of 40)", the entire data restoration is complete.

Now ready to test the API

After the data restoration is complete, you can now test the live API. Try some test requests like these:

  • /metadata
  • /v1/variant/chr7:g.55241707G>T
  • /v1/variant/chr9:g.107620835G>A?fields=dbsnp
  • /v1/query/?q=dbnsfp.genename:BTK
  • /v1/query/?q=chr1:69000-70000
  • /v1/query/?q=_exists_:wellderly AND cadd.polyphen.cat:possibly_damaging&fields=wellderly,cadd.polyphen
